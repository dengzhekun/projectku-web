# AI Ecommerce Maintainer Skill 使用说明

这是一个长期可复用、可持续迭代的 Codex Skill，用来维护和优化“AI 电商平台类项目”。

它不是只给 ProjectKu 用。ProjectKu 只是第一个案例，后续类似项目也可以用这个 Skill 快速接手、分析、优化、测试和部署。

## 1. 这个包里有什么

```text
ai-ecommerce-maintainer-skill-package/
├── 使用说明.md
├── install.ps1
└── skill/
    ├── SKILL.md
    ├── agents/openai.yaml
    └── references/
        ├── architecture-discovery.md
        ├── ai-customer-service.md
        ├── knowledge-base.md
        ├── ecommerce-core.md
        ├── deployment-safety.md
        ├── testing-verification.md
        └── projectku-case-study.md
```

`skill/` 目录就是可以安装到 Codex 的 Skill 本体。

`使用说明.md` 是给人看的说明。

`install.ps1` 是 Windows 一键安装脚本。

## 2. 适合哪些项目

适合这些类型：

- Vue / React / Next.js 电商前端
- Spring Boot / Node.js / Python 后端商城
- 带商品、SKU、价格、库存、购物车、订单、支付、余额、优惠券、评价、售后的项目
- 带 AI 客服、Agent、RAG、LightRAG、GraphRAG、知识库后台的项目
- 需要本地启动、服务器部署、Docker 运维、GitHub 发版、性能测试的项目
- 类似 ProjectKu 的二开商城、AI 客服商城、知识库客服平台

不太适合：

- 完全不涉及电商业务的普通博客/官网
- 纯模型训练项目
- 只有静态页面、没有业务后端的项目

## 3. 它可以帮 AI 做什么

### 3.1 快速接手项目

让 AI 先识别：

- 项目根目录在哪里
- 前端是什么技术栈
- 后端是什么技术栈
- AI 服务在哪里
- 数据库和部署文件在哪里
- 哪些脚本可以启动、测试、打包、部署

避免 AI 一上来就在错误目录改代码。

### 3.2 优化 AI 客服

沉淀了这些经验：

- 商品价格、库存、SKU 要查实时后端 API
- 售后、物流、优惠券规则适合走知识库
- 订单、钱包、个人优惠券必须鉴权
- 低置信度要追问，不要瞎编
- “苹果多少钱”要澄清，“苹果15/iPhone 15”才更像手机型号
- 多版本/多内存/多规格时要列候选或追问

### 3.3 维护知识库和 LightRAG

可以指导 AI 检查：

- 文档有没有导入
- 有没有索引到当前运行态
- embedding 网关是否正常
- LightRAG 是否真的能 query
- 命中日志为空是不是来源追踪问题
- 未命中问题是否应该转成知识库草稿

### 3.4 检查电商核心业务

重点覆盖：

- 商品和 SKU 是否匹配
- 加购物车是否误扣库存
- 结算/支付是否原子扣库存
- 优惠券是否检查门槛和归属
- 余额支付是否防止重复扣款
- 评价和售后是否绑定订单

### 3.5 安全部署

会提醒 AI：

- GitHub 更新不等于服务器更新
- 部署前必须备份
- 不要乱删 Docker volume
- 不要误停同服务器上的其他服务
- 只重启必要容器
- 验证成功后才能说“部署好了”

### 3.6 测试和性能验证

会引导 AI 根据改动选择：

- 前端 build
- 后端单测
- AI 服务 pytest
- AI 客服回归用例
- LightRAG runtime 检查
- Playwright smoke
- k6 / JMeter 性能测试

## 4. 如何安装

在 Windows PowerShell 里进入这个包目录：

```powershell
cd D:\web-main\skill-packages\ai-ecommerce-maintainer-skill-package
.\install.ps1
```

安装后位置：

```text
C:\Users\<你的用户名>\.codex\skills\ai-ecommerce-maintainer
```

如果已经安装过，脚本会覆盖旧版本。

## 5. 如何让 AI 使用

可以直接对 AI 说：

```text
使用 ai-ecommerce-maintainer skill，接手并分析这个 AI 电商项目。
```

或者：

```text
使用 ai-ecommerce-maintainer skill，帮我优化这个类似 ProjectKu 的项目，先识别架构，再给优化方案。
```

也可以针对某个方向：

```text
使用 ai-ecommerce-maintainer skill，检查 AI 客服和 LightRAG 知识库还有哪些可以优化。
```

```text
使用 ai-ecommerce-maintainer skill，帮我做服务器部署前检查，不要直接更新服务器。
```

```text
使用 ai-ecommerce-maintainer skill，检查购物车、库存、订单和余额支付有没有风险。
```

## 6. 怎么持续提升这个 Skill

每次踩坑后，把经验补进去：

- 通用 AI 客服经验：补到 `skill/references/ai-customer-service.md`
- 通用知识库经验：补到 `skill/references/knowledge-base.md`
- 商品/订单/库存/支付经验：补到 `skill/references/ecommerce-core.md`
- 部署和 Docker 经验：补到 `skill/references/deployment-safety.md`
- 测试和性能经验：补到 `skill/references/testing-verification.md`
- ProjectKu 特有经验：补到 `skill/references/projectku-case-study.md`

注意：

- 不要写真实密钥
- 不要写服务器密码
- 不要写一次性聊天废话
- 只沉淀以后还会重复用到的经验

## 7. 给其他 AI 的推荐提示词

把这个包发给其他 AI 后，可以这样说：

```text
请安装并使用这个 ai-ecommerce-maintainer Skill。
它用于维护 AI 电商平台类项目，覆盖前端、后端、AI 客服、RAG/LightRAG、知识库、商品、订单、支付、优惠券、库存、测试和部署。
请先阅读 Skill 的 SKILL.md，再按任务读取 references，不要一次性乱改代码。
```

如果对方不能安装 Skill，也可以让它直接读：

```text
请先阅读 ai-ecommerce-maintainer-skill-package/skill/SKILL.md，
然后根据当前任务读取 skill/references 里的相关文档，
再开始分析或修改项目。
```
