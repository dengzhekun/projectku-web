# 优惠券规则（项目真实来源版）

## 项目内来源
- `back/src/main/java/com/web/service/impl/CouponServiceImpl.java`
- `back/src/main/java/com/web/controller/CouponController.java`
- `back/src/main/java/com/web/service/impl/OrderServiceImpl.java`
- `back/sql/init_db.sql`
- `frontend/src/views/CheckoutView.vue`

## 当前系统支持的优惠券能力
当前项目支持：
- 查询当前用户可用优惠券：`GET /v1/coupons/available`
- 校验优惠券是否可用：`POST /v1/coupons/{code}/check`
- 下单时传入 `couponCode`，后端会再次校验优惠券。

优惠券校验返回：
- `valid`: 是否可用。
- `discount`: 可抵扣金额。
- `reason`: 不可用原因。

## 当前初始化优惠券
初始化 SQL 中给用户 1 发放了优惠券：

- 券码：`NEW300`
- 名称：新客满5000-300
- 类型：`full_reduction`
- 使用门槛：订单金额满 5000 元
- 抵扣金额：300 元
- 状态：`VALID`
- 有效期：初始化时间前 1 天到初始化时间后 30 天

客服回答时可以说：当前项目种子数据里存在 `NEW300`，规则是满 5000 减 300；但用户实际是否拥有这张券，要以当前登录账号的优惠券列表为准。

## 优惠券不可用的真实原因
根据当前后端代码，优惠券不可用主要有以下原因：

1. 优惠券不存在，或不属于当前登录用户。
2. 优惠券状态不是 `VALID`，例如已使用或已过期。
3. 当前时间不在优惠券有效期内。
4. 当前订单金额未达到优惠券最低使用门槛。

如果用户问“为什么我的优惠券不能用”，客服应优先按这四类原因解释。

## 未达到金额时怎么回答
如果订单金额未达到优惠券门槛，系统会判定不可用。客服应该明确告诉用户：这张券当前还不能使用，因为订单金额没有达到门槛。

后台或前端展示时，建议小字提示：

“未达到使用门槛，还差 X 元可用。”

如果无法知道还差多少，就提示：

“未达到该优惠券的最低消费金额，请以结算页展示为准。”

## 下单时的二次校验
当前项目在提交订单时会再次调用优惠券校验。即使前端显示可用，后端仍会二次判断归属、状态、有效期和门槛。

如果优惠券二次校验失败，下单会失败并返回 `COUPON_INVALID`。

## 优惠券和订单金额
当前后端以下单时购物车勾选商品的商品金额计算订单总额，再用该金额校验优惠券门槛。

当前代码没有定义以下复杂规则：
- 多张优惠券叠加。
- 平台券与店铺券叠加顺序。
- 运费是否计入门槛。
- 指定品类或指定商品券。
- 退款后优惠券是否返还。

这些问题不能硬编，客服应引导用户以结算页、券详情或人工客服确认为准。

## 标准客服回答示例
用户问：为什么 NEW300 不能用？

回答：`NEW300` 在当前项目种子数据中是“满 5000 减 300”的优惠券。不能用通常是因为优惠券不属于当前账号、已经使用或过期、还没到有效期，或者订单金额未满 5000 元。请以结算页优惠券提示为准。

用户问：没达到金额的优惠券可以先用吗？

回答：不可以。当前系统会在结算和提交订单时校验门槛，订单金额没有达到最低消费金额时不能使用该优惠券。页面可提示“未达到使用门槛”。
